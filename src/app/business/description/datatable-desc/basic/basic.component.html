表格入门

表格需要一个对象数组作为数据载体，用 p-column 组件定义的列元素。 本例中使用到的 car 接口拥有 vin、brand、year 和 color 四个属性，car实例的集合组成了表格的数据载体。 一般来说表格显示的数据是通过 http 服务从后台获取、在前台解析并展示的。（该示例中使用的是Dummy数据） 


export interface Car {
    vin;
    year;
    brand;
    color;
}



import {Injectable} from 'angular2/core';
import {Http, Response} from 'angular2/http';
import {Car} from '../domain/car';
    
@Injectable()
export class CarService {
    
    constructor(private http: Http) {}

    getCarsSmall() {
        return this.http.get('/showcase/resources/data/cars-small.json')
                    .toPromise()
                    .then(res => <Car[]> res.json().data)
                    .then(data => { return data; });
    }
}

侦查数据变化（Change Detection）

表格使用基于 setter 的数据检查机制，利用 ngDoCheck 将底层数据的变化反映到视图上。 通过配置 immutable 属性，当该属性启用时(默认)，基于 setter 的检测是通过检测数据变化实现的。 如添加或删除一条数据应该创建一个新的数组引用，而不是操纵现有的数组。 因为如果引用没有变的话，angular是不会触发 setter 检查的。 例如，当删除一条数据的时候，应该使用slice而不是splice方法；当添加一条数据的时候，应该使用spread而不是push方法。 另一方面，将 immutable 属性设置为‘false’可以解除上述限制。 使用ngDoCheck 的 IterableDiffers 监听变化而不需要创建一个新的引用。 第一种基于 Setter 的方法更快，可以根据业务需求（自身喜好）自行选择用哪一种方式。 注意，immutable 属性还声明了表格如何处理数据，例如当 immutable 属性启用的时候，对表格进行排序操作不会改变原始数据，而创建一个新的数组进行排序。

模板（Templates）

Filed字段默认外观下是表格中的一个单元格，不过可以通过模板组件改变其展示形式。

【注意】如果想在 p-column 组件中使用 ng-template 的话，必须连同 pTemplate 指令一起使用。pTemplate 指令指定了 ng-template 所属的类型。可能的取值是"header"、"body"以及"footer"。


<p-column field="color" header="Color">
    <ng-template let-col let-car="rowData" let-ri="rowIndex" pTemplate="body">
        <span>{{car[col.field]}}</span>
    </ng-template>
</p-column>
<p-column>
    <ng-template pTemplate="header">
        <button type="button" pButton (click)="selectAllCars()" icon="fa-check"></button>
    </ng-template>
    <ng-template let-car="rowData" pTemplate="body">
        <button type="button" pButton (click)="selectCar(car)" icon="fa-search"></button>
    </ng-template>
</p-column>

ng-template 中可以获得当前数据的下标（index）


    <p-column>
        <ng-template let-car="rowData" let-i="rowIndex" pTemplate="body">

分页（Paginator）

通过将 paginator 属性设置为‘true’可以实现表格的分页效果。

rows 属性表示每页可以展示的数据条数;pageLinks 属性定义分页区域展示多少个可见页面链接。


<p-dataTable [value]="cars" [rows]="10" [paginator]="true">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

可以通过改变 first 属性的值来控制表格的分页效果。first 属性的改变会触发分页效果。这个属性支持数据的双向绑定，所以这个值改变后也能反映到页面上。 下面这个例子就实现了通过JS重置当前页签的效果。 


<p-dataTable [value]="cars" [rows]="10" [paginator]="true" [(first)]="first">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

<button type="button" (click)="reset()" label="Reset"></button>



export class DataTableDemo implements OnInit {

    cars: Car[];
    
    first: number = 0;

    constructor(private carService: CarService) { }

    ngOnInit() {
        this.carService.getCarsSmall().then(cars => this.cars = cars);

排序（Sorting）

一般情况下，如果需要对某一列进行排序操作，只需将其 sortable 属性设置为‘true’即可。用这个属性进行排序的时候使用的是默认字段，也可以通过 sortField 属性设置。


<p-column field="vin" header="Vin" sortable="true"></p-column>

可排序模式下，默认点击某一列的头部就会触发排序操作。如果要实现多列联合排序效果，需要将 sortMode 属性设置为‘multiple’，并且在点击其他列的时候需要借助操作系统的metakey按键。

【注意】metaKey是和系统有关的，在Windows键盘中是Windows键，在Mac中是Cmd键。


<p-dataTable [value]="cars" [sortMode]="multiple">

如果想要表格加载的时候就是按照某种规则排好序的效果，需要在单列排序模式下结合 sortField 和 sortOrder 属性一起使用，示例如下：


<p-dataTable [value]="cars" sortField="year" [sortOrder]="1">
    <p-column field="vin" header="Vin" sortable="true"></p-column>
    <p-column field="year" header="Year" sortable="true"></p-column>
    <p-column field="brand" header="Brand" sortable="true"></p-column>
    <p-column field="color" header="Color" sortable="true"></p-column>
</p-dataTable>

多列联合排序模式下，可以使用存放着多个 SortMeta 对象的 multiSortMeta 数组实现，如下：


<p-dataTable [value]="cars" [multiSortMeta]="multiSortMeta">
    <p-column field="vin" header="Vin" sortable="true"></p-column>
    <p-column field="year" header="Year" sortable="true"></p-column>
    <p-column field="brand" header="Brand" sortable="true"></p-column>
    <p-column field="color" header="Color" sortable="true"></p-column>
</p-dataTable>



this.multiSortMeta = [];
this.multiSortMeta.push({field: 'year', order: 1});
this.multiSortMeta.push({field: 'brand', order: -1});

表格也支持自定义排序功能，如果要实现特殊的排序效果，需要将 sortable 属性设置为‘custom’，并为其绑定一个自定义的函数，在该函数中实现想要的排序效果。


<p-dataTable [value]="cars" [multiSortMeta]="multiSortMeta">
    <p-column field="vin" header="Vin" sortable="true"></p-column>
    <p-column field="year" header="Year" sortable="custom" (sortFunction)="mysort($event)"></p-column>
    <p-column field="brand" header="Brand" sortable="true"></p-column>
    <p-column field="color" header="Color" sortable="true"></p-column>
</p-dataTable>

    //event.field = Field to sort
    //event.order = Sort order

过滤器（Filtering）

在某一列上将其 filter 属性设置为‘true’可以实现数据过滤器的效果，数据过滤有多种匹配模式可选，默认的匹配模式为‘startsWith’。 可以通过改变 filterMatchMode 属性的值改变匹配模式，其他可选值有"contains", "endsWith", "equals" 和 "in"。


<p-column field="vin" header="Vin (startsWith)" [filter]="true" filterPlaceholder="Search"></p-column>
<p-column field="year" header="Year (contains)" [filter]="true" filterMatchMode="contains"></p-column>
<p-column field="brand" header="Brand (startsWith)" [filter]="true"></p-column>
<p-column field="color" header="Color (endsWith)" [filter]="true" filterMatchMode="endsWith"></p-column>

表格也支持全局模式下的过滤设置，而不仅仅是针对某一列。 通常情况下是一个被监听keyup事件的输入框，在输入框的值发生变化的时候，会搜索表格中所有的数据，筛选出满足条件的数据。 


<input #gb type="text" placeholder="Global search">
<p-dataTable [value]="cars" [rows]="10" [globalFilter]="gb">

通常会将输入框作为默认的全局过滤形式，也可以使用模板组件自定义其展现效果。（如下，一个拥有下拉列表的过滤器）切记要为过滤组件绑定change事件的回调函数，以触发过滤效果。


<p-column field="brand" header="Brand (Custom)" [filter]="true" [style]="{'overflow':'visible'}" filterMatchMode="equals">
    <ng-template pTemplate="filter" let-col>
        <p-dropdown [options]="brands" [style]="{'width':'100%'}" (onChange)="dt.filter($event.value,col.field,col.filterMatchMode)" styleClass="ui-column-filter"></p-dropdown>
    </ng-template>
</p-column>

行数据选择（Selection）

表格提供单选和多选两种数据选择模式，可以通过访问 selection 属性获取选择到的行数据。 并且表格为我们提供了 onRowSelect、onRowUnselect 监听事件。 另外可以通过设置 selectionMode 属性的值实现不同的选择效果，如常见的单选按钮或复选框。

单选模式下，selection 属性的值是一个对象的索引。


export class DataTableDemo implements OnInit {

    cars: Car[];
    
    selectedCar: Car;

    constructor(private carService: CarService) { }

    ngOnInit() {
        this.carService.getCarsSmall().then(cars => this.cars = cars);
    }
}



<p-dataTable [value]="cars" selectionMode="single" [(selection)]="selectedCar" dataKey="vin">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

多选模式下，selection 属性的值是一个数组。可以借助键盘上的 shift 按键实现常用的连续多选效果。

    <p-column selectionMode="multiple"></p-column>
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>

当程序处理行选中时，默认情况表格会通过对比 selection 属性的值和源数据得到结果，这在拥有庞大的数据集合且没有使用分页的情况下会产生性能问题，可以的话最好使用 dataKey 属性为每一行分配一个不重复的id， 这样使用数组内部的一个map实例从而避免对整个数组进行遍历。 如果无法使用 dataKey 为每一行指定id，也可以考虑对 compareSelectionBy 属性使用‘equals’代替默认的‘deepEquals’，由于deepEquals会检查对象的所有属性，所以其效率是比较低的。

调节列宽（Column Resize）

可以通过将 resizableColumns 属性设置为‘true’从而实现拖拽表格改变其尺寸的效果。 columnResizeMode 属性有两种取值："fit" 和 "expand"。 默认模式是 fit，这种模式下，表格的整体宽度不会随着个别列宽度的改变而变化。 expand 模式下，表格的整体宽度会随着个别列宽度的改变而改变。 两种模式下都提供了 onColumnResize 回调方法，将宽度发生变化的列的header当做参数传递给该函数。 


<p-dataTable [value]="cars" [resizableColumns]="true">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

值得注意的是，当你需要改变某列宽度的时候，不能设置该列的宽度为固定的px值，应该使用百分比代替。使用固定值px的话，该列是不可以改变宽度的。


<p-dataTable [value]="cars" [resizableColumns]="true">
    <p-column field="vin" header="Vin" [style]="{'width':'20%'}"></p-column>
    <p-column field="year" header="Year" [style]="{'width':'30%'}"></p-column>
    <p-column field="brand" header="Brand" [style]="{'width':'15%'}"></p-column>
    <p-column field="color" header="Color" [style]="{'width':'35%'}"></p-column>
</p-dataTable>

数据导出（Data Export）

通过调用 exportCSV 方法，可以导出表格中的数据，最终导出的是.CSV文件。

通过设置 scrollWidth 和 scrollHeight 的值可以触发表格的滚动条。这两个属性的值可以是固定的px值也可以是百分比。


<p-dataTable [value]="cars" [scrollable]="true" scrollHeight="200px">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

水平方向的滚动条，可以通过设置某一列的 frozen 属性为‘true’从而实现固定该列的效果。


<p-dataTable [value]="cars" [scrollable]="true" scrollHeight="200px" frozenWidth="100px" scrollWidth="600px">
    <p-column field="vin" header="Vin" frozen="true"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

此外，滚动条提供在滚动的过程中动态加载数据的功能。

【说明】一般来说，纵向滚动条和表格的分页功能只要一种即可，不需要同时使用。

数据懒加载（Lazy Loading）

数据懒加载适用于处理大量数据集合的情况。通过配置回调函数，可以实现分页、排序和过滤数据时，每次只加载一部分数据，而不是将整个数据集合都加载出来。 要实现数据懒加载，首先需要将 lazy 属性设置为‘true’。接下来，为 onLazyLoad 属性提供一个回调事件，在该函数中实现自动请求、加载数据的功能。 该函数接受一个event对象，里面存放着需要展现的数据的相关信息。值得注意的是，由于初次加载的时候，只加载了部分数据，而不是全部的数据集合，所以需要为 totalRecords 指定一个总数据条数。 这样才能正确的实现分页的效果（不知道数据总条数的话是无法实现分页效果的，因为都不知道能分多少页呀） 


<p-dataTable [value]="cars" [scrollable]="true" [lazy]="true" (onLazyLoad)="loadData($event)" [totalRecords]="totalRecords">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>

子元素中的滚动条（Overlays in Cells）

默认情况下，表格禁止其子元素中出现滚动条。某些特殊情况下，例如使用 ng-template 实现子元素是一个下拉列表的效果，这种情况下条目比较多的时候就需要显示出滚动条，可以通过设置 style 属性的 overflow 为‘visible’来实现。


<p-column field="color" [style]="{'overflow':'visible'}">
    <ng-template let-col let-car="rowData">
        <p-dropdown></p-dropdown>
    </ng-template>
</p-column>

数据加载效果（Loading Status）

如果在表格加载数据时需要展示一个加载动作的效果，可以将其 loading 属性设置为‘loading’。

通过配置 loadingIcon 的值，可以展现不同的加载按钮。


<p-dataTable [value]="cars" [loading]="loading" loadingIcon="fa-spinner">
    <p-column field="vin" header="Vin"></p-column>
    <p-column field="year" header="Year"></p-column>
    <p-column field="brand" header="Brand"></p-column>
    <p-column field="color" header="Color"></p-column>
</p-dataTable>



export class DataTableDemo implements OnInit {

    loading: boolean;

    cars: Car[];

    constructor(private carService: CarService) { }

    ngOnInit() {
        this.loading = true;
        setTimeout(() => {
            this.carService.getCarsSmall().then(cars => this.cars = cars);
            this.loading = false;
        }, 1000);
    }
}

几条性能优化的tips（Performance Tips Status）

  • 1.使用表格的 selection 属性实现选择数据效果时，比较对象时避免使用深度比较（deep checking）；
  • 2.利用 rowTrackBy 属性的 ngForTrackBy 来阻止不必要的Dom操作；
  • 3.加载大数据集的时候，优先使用懒加载。
