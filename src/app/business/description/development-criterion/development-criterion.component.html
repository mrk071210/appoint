常用命令

【简介】angular-cli提供了一些便捷的命令，用于一键生成模块、组件、服务、指令等。（强烈推荐用命令行的方式生成模块、组件等，不推荐手动创建文件夹以及文件） 详情参考

【使用说明】“ng g” 是 “ng generate” 的简写形式，使用两者任意一种即可；命令中的第三个词是需要创建的类型，常见的类型如上表；第四个词是需要创建的命名。

【Demo】例如，我需要在 “src/app/business” 文件夹下生成一个名为 “component-a” 的组件。

  • step1：cd到目标文件夹
  • step2：执行 “ng g component component-a” 命令
【注意】命名方式：小写、单词链接用中划线；禁止手动更改通过上述命令行生成的文件名。

项目配置文件（.angular-cli.json）

【简介】Angular-CLI 是 Google Angular 核心团队提供的开发工具，集成了大量的 NodeJS 模块，例如：webpack，karma，asmine，typescript 编译器等。只要一条命令就能帮你自动创建项目的目录结构、自动帮你生成 Component 骨架代码并自动添加依赖、自动热加载代码等等。

打包配置文件（package.json）

【简介】npm配置为文件，其中列出了项目中使用到的第三方依赖包，也可以在这个文件中添加自己的自定义脚本。执行“npm serve”命令时，会自动下载该文件中配置的依赖包至 node_modules 文件夹下。

【特别说明】开发者并不需要关心这个文件里面的内容，唯一需要注意的是我们封装了一些 angular-cli 的命令，位于 scripts 对象中。

【Demo】以下两者是等价的：


// angular-cli 指令
ng g module module-name



// npm 指令
npm run create module module-name
